java学习笔记-3-标识符和数据类型

[TOC]

标识符

  1. 标识符是由字母、数字、下划线（_）和美元符号（$）构成的字符串。
  2. 标识符必须以字母、下划线（_）或美元符号($)开头，不能用以数字开头。
  3. 标识符不能是 java 的关键字。
  4. 标识符可以是任意长度。
  5. 区分大小写。

数据类型

java 共有 8 种数据类型，其中包括 4 种整数类型（byte、short、int、long），两种浮点数据类型（float、double），一种字符类型（char，用于 Unicode 编码中的字符），以及用于表示真假的布尔类型（boolean）。

整数类型的长度和范围

类型 位 字节 取值范围
byte 8 1 -128~+127
short 16 2 -215~+215-1
int 32 4 -231~+231-1
long 64 8 -263~+263-1

当整数表达式包含 byte、short、int 及字面量数字时，在进行运算之前所有表达式的类型将被提升到整型。

整数的二进制表示（0x表示十六进制，0b表示二进制），例如“byte aByte = （byte） 0b00100001”；

利用下划线作为间隔，使所写的数值文字更加清楚，例如“int billion = 1_000_000_000”。

浮点类型的长度和范围

类型 位 字节 取值范围
float 32 4 -3.4E-38~+3.4E+38
double 64 8 -1.7E-308~+1.7E+308

很多情况下，float 类型的精度时不够，所以大部分应用程序中使用的都是 double 类型，float 类型只有在很少的场合。

浮点字面值默认被定义为 double（64位） ，因此，如果要把浮点字面值赋给 float（32位） 类型的变量，必须把后缀 F 或 f 附加到该数字上。

存储字符的数据结构是 char ，它包含单个 16 位的 Unicode 字符。字面值被记为单引号内的单个字符。如 char a = 'a';

把整数赋给 char 类型变量的范围为 0~65535，也可用 Unicode 表示法，如 “char a = ‘\u0000’”。

布尔类型 boolean 没有范围，他得值只能是 true 或 false，并不像 C 语言中可以用数字表示true 或 false。

数据类型之间的转换

  1. 数据类型之间的自动转换

    • 如果两个操作数中有一个为 double 型，另一个操作数就会自动转换为 double 型。
    • 否则，如果其中一个是 float 型，另一个操作数就会自动转换为 float 型。
    • 否则，如果其中一个是 long 型，另一个操作数就会自动转换为 long 型。
    • 否则，两个操作数都将转换为 int 型。

  2. 数据类型之间的强制转换

    强制类型转换的语法格式是在圆括号中给出想要转换的目标类型，后面紧跟着待转换的变量名。

java学习笔记-2-java编程规范

[TOC]

命名

  1. 包命名：按照域名的范围从大到小依次列出，恰好和Internet上域名的命名规则相反。包名由一组以“.”连接的标识符构成，通常，第一个标识符为符合网络域名的两个或三个英文小写字母，例如org.javaTest。

  2. 类、接口命名：类的名字必须由大写字母开头，且单词的其他字母均小写。如果类名由多个单词组成，则每个单词的首字母均大写，有缩写时，缩写词每个字母均大写，且类名应尽量选择名词。

  3. 方法名：方法名的第一个单词应以小写字母开头，后面的单词则用大写字母开头，可以为动词或“动词+名词”的组合。

    • 设置/获取某个值的Method，应该遵循setV/getV规范，例如“getData（）;”。

    • 返回长度的Method，应该命名为length，例如“length（）；”。

    • 测试某个布尔值得Method，应该命名为isVS，例如“isReady（）；”。

    • 将对象转换为某个特定类型的Method，应该命名为toF，例如“toOracleFormat（）；”。

  4. 变量名：局部变量、方法参数的命名应该采用首字母小写，其他变量字母大写的方式；final static 变量的名字应该都大写，并指出完整意义，且以下划线分隔这些单词；常量取代数字、固定字符串。

索引变量：i、j、k等只能作为小型循环的循环索引变量。

逻辑变量：避免用 flag 来命名状态变量、用 is 来命名逻辑变量。

注释
// 为行注释，即注释只能写在一行；

/* ... */可以将一段比较长的注释包含进去，即支持多行注释；

以 /** 开始，以 */ 结束的自动生成专业水准的文档注释，它可以将代码和文档保存在一个地方。

  1. 类注释：必须放在 import 语句之后、类定义之前，可以生成类的相关文档。

  2. 方法注释： 每一个方法的注释都必须放在所描述的方法之前。

  3. 域注释：只需要对公共域（通常指的是静态变量）建立文档。

  4. 通用注释：

  • @author name 这个标记将产生一个作者条目。

  • @version text 这个标记将产生一个版本条目。

  • @since text 这个标记将产生一个始于条目。

  • @deprecate text 这个标记将对类，方法或变量添加一个不再使用的注释。
