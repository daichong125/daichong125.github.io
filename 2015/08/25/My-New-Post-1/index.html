java学习笔记-2-java编程规范

[TOC]

命名

  1. 包命名：按照域名的范围从大到小依次列出，恰好和Internet上域名的命名规则相反。包名由一组以“.”连接的标识符构成，通常，第一个标识符为符合网络域名的两个或三个英文小写字母，例如org.javaTest。

  2. 类、接口命名：类的名字必须由大写字母开头，且单词的其他字母均小写。如果类名由多个单词组成，则每个单词的首字母均大写，有缩写时，缩写词每个字母均大写，且类名应尽量选择名词。

  3. 方法名：方法名的第一个单词应以小写字母开头，后面的单词则用大写字母开头，可以为动词或“动词+名词”的组合。

    • 设置/获取某个值的Method，应该遵循setV/getV规范，例如“getData（）;”。

    • 返回长度的Method，应该命名为length，例如“length（）；”。

    • 测试某个布尔值得Method，应该命名为isVS，例如“isReady（）；”。

    • 将对象转换为某个特定类型的Method，应该命名为toF，例如“toOracleFormat（）；”。

  4. 变量名：局部变量、方法参数的命名应该采用首字母小写，其他变量字母大写的方式；final static 变量的名字应该都大写，并指出完整意义，且以下划线分隔这些单词；常量取代数字、固定字符串。

索引变量：i、j、k等只能作为小型循环的循环索引变量。

逻辑变量：避免用 flag 来命名状态变量、用 is 来命名逻辑变量。

注释
// 为行注释，即注释只能写在一行；

/* ... */可以将一段比较长的注释包含进去，即支持多行注释；

以 /** 开始，以 */ 结束的自动生成专业水准的文档注释，它可以将代码和文档保存在一个地方。

  1. 类注释：必须放在 import 语句之后、类定义之前，可以生成类的相关文档。

  2. 方法注释： 每一个方法的注释都必须放在所描述的方法之前。

  3. 域注释：只需要对公共域（通常指的是静态变量）建立文档。

  4. 通用注释：

  • @author name 这个标记将产生一个作者条目。

  • @version text 这个标记将产生一个版本条目。

  • @since text 这个标记将产生一个始于条目。

  • @deprecate text 这个标记将对类，方法或变量添加一个不再使用的注释。
